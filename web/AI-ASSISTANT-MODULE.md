# AI 智能助手模块文档

## 📋 概述

AI 智能助手模块将人工智能与需求管理系统深度结合，提供 8 大核心功能，帮助团队**提高效率、降低成本**。

## 🎯 核心功能

### 1. AI 需求生成助手
**API**: `POST /api/ai/generate-requirement`

**场景**: 用户输入简短描述，AI 自动生成完整需求文档

**请求示例**:
```json
{
  "userInput": "用户登录功能"
}
```

**响应示例**:
```json
{
  "success": true,
  "message": "需求生成成功",
  "data": {
    "result": "完整的需求文档内容...",
    "type": "requirement"
  }
}
```

**输出内容**:
- 功能概述（100-200 字）
- 用户故事
- 功能详述（核心功能点 3-5 个）
- 验收标准（3-5 条，可量化、可测试）
- 边界情况（异常场景、边界值）
- 非功能需求（性能、安全、兼容性）

---

### 2. AI 测试用例生成
**API**: `POST /api/ai/generate-testcase`

**场景**: 根据需求描述自动生成测试用例

**请求示例**:
```json
{
  "requirementDoc": "需求文档内容...",
  "requirementId": "REQ-001"
}
```

**响应示例**:
```json
{
  "success": true,
  "message": "测试用例生成成功",
  "data": {
    "result": "测试用例内容...",
    "type": "testcase",
    "requirementId": "REQ-001"
  }
}
```

**输出内容**:
- 正常场景测试（Happy Path）
- 异常场景测试（输入验证、边界值、错误处理）
- 边界值分析（最小值/最大值/空值/特殊字符）
- 测试用例模板（ID、名称、优先级、步骤、预期结果）

---

### 3. AI 需求审查
**API**: `POST /api/ai/review-requirement`

**场景**: 自动检查需求质量

**请求示例**:
```json
{
  "requirementText": "需求文档内容...",
  "requirementId": "REQ-001"
}
```

**审查维度**:
1. **清晰度检查**: 识别模糊词汇（"可能"、"大概"、"等"）
2. **完整性检查**: 功能概述、用户故事、验收标准
3. **可测试性检查**: 验收标准是否可量化
4. **一致性检查**: 需求内部是否自洽
5. **可行性检查**: 技术实现是否可行

**输出内容**:
- 整体评分（1-5 星）
- 发现的问题（带严重程度）
- 模糊词汇检测报告
- 改进建议
- 验收标准评估

---

### 4. AI 缺陷自动分类
**API**: `POST /api/ai/classify-defect`

**场景**: 根据缺陷描述自动分类和定级

**请求示例**:
```json
{
  "defectDescription": "登录页面在移动端显示异常...",
  "defectId": "DEF-001",
  "includeHistory": true
}
```

**输出内容**:
- 缺陷类型（功能/界面/性能/安全/兼容性/数据/文档）
- 严重程度（致命/严重/一般/轻微）
- 优先级建议（P0/P1/P2/P3）
- 推荐指派角色（前端/后端/测试/产品）
- 关键信息提取（涉及模块、触发条件、影响范围）
- 相似缺陷关联（如有历史数据）

---

### 5. AI 任务智能分配
**API**: `POST /api/ai/assign-task`

**场景**: 根据任务内容和团队成员能力自动分配

**请求示例**:
```json
{
  "taskDescription": "实现用户注册功能...",
  "taskId": "TASK-001"
}
```

**输出内容**:
- 任务分析（所需技能、复杂度、预估工时）
- 人员匹配分析表
- 推荐负责人及理由
- 备选负责人
- 协作建议
- 风险提示
- 跟进建议（里程碑）

---

### 6. AI 智能搜索
**API**: `POST /api/ai/search`

**场景**: 自然语言搜索需求/用例/缺陷

**请求示例**:
```json
{
  "query": "登录相关的测试用例",
  "filters": {
    "projectId": "PROJ-001"
  }
}
```

**输出内容**:
- 搜索意图分析（用户真实需求、关键搜索词）
- 搜索结果（需求、用例、缺陷、任务）
- 搜索结果按相关性排序
- 搜索建议（相关搜索、细化条件、扩展搜索）

---

### 7. AI 变更影响分析
**API**: `POST /api/ai/impact-analysis`

**场景**: 需求变更时自动分析影响范围

**请求示例**:
```json
{
  "changeDescription": "修改登录密码规则，从 6 位改为 8-16 位...",
  "requirementId": "REQ-001"
}
```

**输出内容**:
- 变更概述（内容、原因、紧急程度）
- 影响范围（直接影响、间接影响）
- 需要修改的文档/用例/代码模块
- 工作量评估（需求、开发、测试、文档）
- 风险评估（技术、进度、质量）
- 建议措施
- 变更流程建议

---

### 8. AI 项目风险预测
**API**: `POST /api/ai/risk-prediction`

**场景**: 基于历史数据预测项目风险

**请求示例**:
```json
{
  "projectId": "PROJ-001"
}
```

**分析维度**:
- 需求稳定性分析（变更频率、趋势）
- 进度风险分析（完成率、延期情况）
- 质量风险分析（缺陷密度、修复率）
- 资源风险分析（人员负载、技能匹配）

**输出内容**:
- 整体风险评级（高/中/低）
- 各维度风险等级
- 风险预警（🔴高风险、🟡中风险）
- 风险缓解建议
- 监控指标建议

## 🔧 技术实现

### 文件结构
```
web/
├── ai-assistant.js      # AI 助手核心逻辑
├── ai-prompts.js        # AI 提示词模板库
├── ai-assistant.html    # AI 助手前端页面
├── api-server.js        # API 服务器（已集成 AI 路由）
└── middleware.js        # 中间件（已添加 AI 错误码）
```

### AI 集成方式

#### 1. 本地 AI 模型（优先）
- **服务**: Ollama
- **地址**: http://localhost:11434
- **模型**: qwen2.5:7b（可配置）
- **优势**: 免费、离线、数据隐私

#### 2. 云端 API（备用）
- **服务**: 阿里云百炼（DashScope）
- **模型**: qwen-plus
- **配置**: 环境变量 `DASHSCOPE_API_KEY`
- **优势**: 稳定、高质量

#### 3. 降级处理
- AI 服务不可用时自动返回模板响应
- 确保功能可用性，不影响用户体验

### 缓存机制
- **类型**: 内存缓存（Map）
- **TTL**: 1 小时
- **最大容量**: 1000 条
- **目的**: 减少重复调用，提高响应速度

### 错误处理
- 统一的错误码（10xxx - AI 服务，11xxx - 搜索）
- 详细的错误日志
- 友好的用户提示

## 🚀 部署配置

### 1. 安装 Ollama（推荐）
```bash
# 安装 Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 拉取模型
ollama pull qwen2.5:7b

# 启动服务
ollama serve
```

### 2. 配置云端 AI（可选）
```bash
# 设置环境变量
export DASHSCOPE_API_KEY="your-api-key"
```

### 3. 启动服务
```bash
cd /home/admin/.openclaw/workspace/web
node api-server.js
```

### 4. 验证 AI 状态
```bash
curl http://localhost:8001/api/ai/status
```

## 🐛 常见问题

### Q1: AI 服务不可用
**A**: 检查 Ollama 服务是否启动，或配置云端 API Key

### Q2: 响应速度慢
**A**: 使用更小的模型，或切换到云端 API

### Q3: 生成质量不佳
**A**: 优化输入描述，提供更详细的上下文

### Q4: 缓存如何清理
**A**: 重启服务自动清理，或手动清空 ai-cache.json
