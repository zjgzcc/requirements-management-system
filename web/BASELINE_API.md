# 基线版本管理模块 API 文档

## 概述

基线版本管理模块提供完整的版本快照、对比、回退功能，支持需求和测试用例的版本控制。

## 数据模型

### Baseline 对象

```json
{
  "id": "string",              // 基线唯一 ID
  "projectId": "string",        // 所属项目 ID
  "type": "requirement|testcase", // 基线类型
  "name": "string",             // 基线名称
  "version": "string",          // 版本号 (格式：V1.0, V1.1, V2.0)
  "status": "active|archived|superseded", // 基线状态
  "items": [],                  // 包含的项列表
  "createdAt": "ISO8601",       // 创建时间
  "createdBy": "string",        // 创建人
  "archivedAt": "ISO8601"       // 归档时间 (仅归档基线)
}
```

### Item 对象 (基线中的项)

```json
{
  "id": "string",               // 项 ID
  "reqId": "string",            // 需求 ID 或 用例 ID (如 REQ-1, TC-1)
  "description": "string",      // 描述信息
  "snapshot": {}                // 完整的项快照 (包含所有字段)
}
```

## API 接口

### 1. 创建基线

**POST** `/api/baselines`

创建新的基线版本，自动递增版本号。

#### 请求参数

```json
{
  "projectId": "string",        // 必需，项目 ID
  "type": "requirement|testcase", // 必需，基线类型
  "name": "string",             // 必需，基线名称
  "version": "string",          // 可选，自定义版本号 (默认自动生成)
  "itemIds": ["string"]         // 必需，要包含的项 ID 列表
}
```

#### 响应示例

```json
{
  "success": true,
  "message": "基线创建成功",
  "data": {
    "id": "bl_xxx",
    "projectId": "proj_xxx",
    "type": "requirement",
    "name": "需求基线 V1",
    "version": "V1.1",
    "status": "active",
    "items": [...],
    "createdAt": "2026-03-17T11:00:00.000Z",
    "createdBy": "admin"
  },
  "versionInfo": {
    "autoGenerated": true,
    "previousVersion": "V1.0"
  }
}
```

#### 版本号规则

- 如果未指定版本号，系统自动查找该项目该类型的最新版本并递增
- 版本格式：`V{major}.{minor}` (如 V1.0, V1.1, V2.0)
- 首次创建默认为 V1.0
- 递增规则：V1.0 → V1.1 → V1.2 → V2.0

### 2. 获取基线列表

**GET** `/api/baselines`

获取基线列表，支持多种筛选和排序选项。

#### 查询参数

| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| projectId | string | 项目 ID 筛选 | - |
| type | string | 类型筛选 (requirement/testcase) | - |
| status | string | 状态筛选 (active/archived/superseded) | - |
| sortBy | string | 排序字段 (createdAt/version/name) | createdAt |
| sortOrder | string | 排序方向 (asc/desc) | desc |

#### 响应示例

```json
{
  "success": true,
  "data": [
    {
      "id": "bl_xxx",
      "version": "V2.0",
      "name": "需求基线 V2",
      "type": "requirement",
      "status": "active",
      "createdAt": "2026-03-17T11:00:00.000Z",
      "items": [...]
    }
  ],
  "stats": {
    "total": 5,
    "byType": {
      "requirement": 3,
      "testcase": 2
    },
    "byStatus": {
      "active": 4,
      "archived": 1,
      "superseded": 0
    }
  }
}
```

### 5. 基线对比

**GET** `/api/baselines/compare?baselineId1=:id1&baselineId2=:id2`

比较两个基线版本的差异。

#### 查询参数

| 参数 | 类型 | 说明 |
|------|------|------|
| baselineId1 | string | 第一个基线 ID (基准版本) |
| baselineId2 | string | 第二个基线 ID (对比版本) |

#### 响应示例

```json
{
  "success": true,
  "data": {
    "baseline1": {
      "id": "bl_1",
      "version": "V1.0",
      "name": "需求基线 V1"
    },
    "baseline2": {
      "id": "bl_2",
      "version": "V2.0",
      "name": "需求基线 V2"
    },
    "added": [
      {
        "id": "req_new",
        "reqId": "REQ-9",
        "description": "新增需求",
        "snapshot": {...}
      }
    ],
    "removed": [],
    "modified": [
      {
        "id": "req_1",
        "reqId": "REQ-1",
        "description": "已修改需求",
        "oldSnapshot": {...},
        "newSnapshot": {...},
        "changes": [
          {
            "field": "description",
            "oldValue": "原描述",
            "newValue": "新描述"
          }
        ]
      }
    ],
    "unchanged": [...]
  },
  "stats": {
    "added": 1,
    "removed": 0,
    "modified": 2,
    "unchanged": 7,
    "total1": 10,
    "total2": 11,
    "changeRate": "30.00%"
  },
  "summary": {
    "hasChanges": true,
    "totalChanges": 3,
    "stabilityScore": 70
  }
}
```

#### 对比说明

- **added**: 在 baseline2 中新增的项
- **removed**: 在 baseline2 中删除的项
- **modified**: 内容发生变化的项 (包含详细变更字段)
- **unchanged**: 未变化的项
- **changeRate**: 变更率 (变更项数 / baseline1 总项数)
- **stabilityScore**: 稳定性评分 (0-100, 越高越稳定)

### 6. 基线回退

**POST** `/api/baselines/:id/restore`

将当前数据恢复到指定基线版本。

#### 请求参数

```json
{
  "createBackup": true  // 可选，是否创建备份基线 (默认 true)
}
```

#### 响应示例

```json
{
  "success": true,
  "message": "成功恢复 10 项",
  "data": {
    "baselineId": "bl_xxx",
    "version": "V1.0",
    "name": "需求基线 V1",
    "type": "requirement",
    "restoredAt": "2026-03-17T12:00:00.000Z",
    "stats": {
      "total": 10,
      "updated": 8,
      "restored": 2,
      "hasChanges": 5,
      "backupCreated": true
    },
    "backup": {
      "id": "bl_backup",
      "name": "需求基线 V1 - 回退前备份",
      "version": "V1.0-backup"
    }
  }
}
```

#### 回退说明

- **updated**: 更新的现有项数量
- **restored**: 恢复的缺失项数量
- **hasChanges**: 实际发生变化的项数量
- **backupCreated**: 是否创建了备份基线
- 备份基线自动标记为 `archived` 状态

### 7. 基线归档

**POST** `/api/baselines/:id/archive`

将基线标记为已归档状态。

#### 响应示例

```json
{
  "success": true,
  "message": "基线已归档",
  "data": {
    "id": "bl_xxx",
    "version": "V1.0",
    "name": "需求基线 V1",
    "status": "archived",
    "archivedAt": "2026-03-17T12:00:00.000Z"
  }
}
```

---

### 8. 删除基线

**DELETE** `/api/baselines/:id`

删除指定基线。

#### 限制

- 不能删除某个项目某类型的最后一个基线
- 建议先归档再删除

#### 响应示例

```json
{
  "success": true,
  "message": "基线已删除",
  "data": {
    "deletedBaseline": {
      "id": "bl_xxx",
      "version": "V1.0",
      "name": "需求基线 V1"
    }
  }
}
```

---

### 9. 基线导出

**GET** `/api/baselines/:id/export`

导出基线为 HTML 格式 (可用于 PDF 转换)。

#### 响应

- Content-Type: `text/html; charset=utf-8`
- 自动下载 HTML 文件

## 使用场景

### 场景 1: 版本发布

1. 完成一轮需求评审后，创建基线 V1.0
2. 后续变更时创建 V1.1, V1.2 等小版本
3. 重大变更时创建 V2.0 等大版本

### 场景 2: 版本对比

1. 选择两个基线版本进行对比
2. 查看新增、删除、修改的项
3. 评估变更影响范围

### 场景 3: 版本回退

1. 发现当前版本有问题
2. 选择历史基线版本进行回退
3. 系统自动创建回退前备份
4. 数据恢复到历史状态

### 场景 4: 基线管理

1. 定期归档旧基线
2. 保持活跃基线数量合理
3. 通过状态筛选快速定位基线

---

## 错误码

| 错误码 | HTTP 状态码 | 说明 |
|--------|------------|------|
| 400 | Bad Request | 参数不完整或格式错误 |
| 404 | Not Found | 基线不存在 |
| 400 | Bad Request | 不能删除最后一个基线 |

---

## 最佳实践

1. **命名规范**: 基线名称应包含版本信息和用途 (如 "Sprint 1 需求基线 V1.0")
2. **定期归档**: 将已发布或被替代的基线标记为 archived
3. **回退前备份**: 回退时保持 createBackup=true (默认)
4. **版本对比**: 重大变更前进行版本对比评估影响
5. **注释记录**: 在创建基线时记录详细的变更说明
