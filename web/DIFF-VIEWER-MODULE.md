# 版本差异对比模块

## 概述

版本差异对比模块提供了可视化的版本对比功能，支持需求、用例和基线的并排对比，高亮显示新增、删除和修改的内容。

## 功能特性

### 1. 可视化差异显示

- ✅ **并排对比视图**：同时显示旧版本和新版本
- ✅ **差异高亮**：
  - 🟢 绿色：新增内容
  - 🔴 红色：删除内容
  - 🟡 黄色：修改内容
  - ⚪ 灰色：未变更内容
- ✅ **字段级差异检测**：精确识别每个字段的变化
- ✅ **差异统计**：实时显示变更数量和影响范围

### 2. 对比模式

- 📋 **需求版本对比**：对比不同基线版本的需求
- 🧪 **用例版本对比**：对比不同基线版本的测试用例
- 📊 **基线版本对比**：对比基线配置本身
- 🔧 **自定义版本对比**：支持任意两个版本的对比

### 3. 差异报告

- 📈 **差异统计报告**：包含新增、删除、修改的统计信息
- 🔍 **变更影响分析**：字段级变更详情
- 📄 **导出差异报告**：
  - Word (.docx) 格式
  - PDF (.pdf) 格式
  - JSON (.json) 格式
- 💬 **差异评论功能**：为每个变更添加评论

### 4. 交互功能

- ⚡ **快速跳转**：一键定位到差异位置
- 🔍 **过滤差异类型**：
  - 只看新增
  - 只看删除
  - 只看修改
  - 只看未变更
  - 全部显示
- ✓ **接受/拒绝变更**：对变更进行审批操作
- 💡 **差异合并建议**：智能推荐合并策略

## 技术实现

### 文件结构

```
web/
├── diff-viewer.html      # 差异对比页面
├── diff-viewer.js        # 差异渲染逻辑
├── diff-viewer.css       # 差异样式
└── api-server.js         # 新增差异对比 API
```

### API 接口

#### 1. 需求差异对比

```
GET /api/diff/requirement/:id1/:id2
```

**参数**:
- `id1`: 旧版本基线 ID
- `id2`: 新版本基线 ID

**响应**:
```json
{
  "success": true,
  "diff": {
    "baseline1": { "id": "...", "version": "V1.0", "name": "..." },
    "baseline2": { "id": "...", "version": "V2.0", "name": "..." },
    "added": [],
    "removed": [],
    "modified": [],
    "unchanged": [],
    "fieldChanges": [],
    "stats": {
      "added": 0,
      "removed": 0,
      "modified": 0,
      "unchanged": 0,
      "total": 0
    }
  }
}
```

#### 2. 用例差异对比

```
GET /api/diff/testcase/:id1/:id2
```

**参数**:
- `id1`: 旧版本基线 ID
- `id2`: 新版本基线 ID

#### 3. 基线版本对比

```
GET /api/diff/baseline/:id1/:id2
```

**参数**:
- `id1`: 基线 1 ID
- `id2`: 基线 2 ID

#### 4. 生成差异报告

```
POST /api/diff/report
```

**请求体**:
```json
{
  "mode": "requirement",
  "version1": "baseline_id_1",
  "version2": "baseline_id_2",
  "diff": { ... },
  "format": "word"
}
```

**格式选项**:
- `word`: Word 文档 (.docx)
- `pdf`: PDF 文档 (.pdf)
- `json`: JSON 数据 (.json)

## 使用指南

### 基本使用流程

1. **访问对比页面**
   - 打开 `http://localhost:8001/diff-viewer.html`

2. **选择对比模式**
   - 点击顶部的模式标签：需求对比 / 用例对比 / 基线对比

3. **选择版本**
   - 从下拉列表中选择旧版本
   - 从下拉列表中选择新版本

4. **开始对比**
   - 点击"🔍 开始对比"按钮

5. **查看结果**
   - 查看并排对比视图
   - 查看差异统计信息

6. **过滤差异**
   - 使用过滤器按钮查看特定类型的变更

7. **导出报告**
   - 点击底部的导出按钮选择导出格式

### 高级功能

#### 字段级差异查看

修改的项目会显示字段级对比：
- 左侧显示旧值（带删除线）
- 右侧显示新值（高亮）

#### 添加评论

1. 点击项目右上角的"💬 评论"按钮
2. 输入评论内容
3. 点击发送

#### 接受/拒绝变更

- **接受变更**：点击"✓ 接受"按钮，确认该变更
- **拒绝变更**：点击"✗ 拒绝"按钮，撤销该变更

## 性能优化

1. **懒加载**：仅加载可见的差异项
2. **虚拟滚动**：大量数据时启用虚拟滚动
3. **缓存机制**：对比结果缓存在内存中
4. **增量更新**：支持增量刷新对比结果

## 扩展性

### 添加新的对比类型

### 自定义导出格式

## 已知限制

1. **PDF 导出**：需要额外的 PDF 生成库支持
2. **大文件对比**：超过 1000 项时性能可能下降
3. **实时协作**：暂不支持多人同时编辑评论

## 故障排除

### 问题：对比结果为空

**解决方案**：
1. 检查两个基线是否存在
2. 确认基线类型匹配
3. 查看服务器日志

### 问题：导出功能失败

**解决方案**：
1. 检查是否有差异数据
2. 确认浏览器允许下载
3. 查看控制台错误信息

### 问题：样式显示异常

**解决方案**：
1. 清除浏览器缓存
2. 检查 CSS 文件是否正确加载
3. 确认浏览器兼容性
