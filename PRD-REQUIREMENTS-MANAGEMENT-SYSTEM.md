## 1. 产品概述

### 1.1 产品定位

**医疗器械行业专业级需求管理系统**

- 基于 V 模型的需求追溯管理
- 支持医疗器械行业合规要求
- 商用级 SaaS 系统

### 1.2 目标用户

| 用户类型 | 典型场景 | 核心需求 |
|---------|---------|---------|
| **医疗器械企业** | 产品研发过程管理 | 需求追溯、合规审计 |
| **项目管理员** | 项目规划与监控 | 进度跟踪、团队管理 |
| **研发工程师** | 需求编写与维护 | 高效编辑、版本管理 |
| **测试工程师** | 测试用例执行 | 用例管理、结果记录 |
| **质量管理人员** | 合规审计 | 追溯矩阵、报告导出 |
| **外部访客** | 客户/审计人员 | 只读查看、报告审阅 |

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| **行业专注** | 专为医疗器械行业设计，支持 V 模型 |
| **灵活定制** | 用户自定义需求集合，不预设业务概念 |

## 2. 设计原则

### 2.1 极简设计原则（核心）

**设计理念：简约、专注、高效**

本系统遵循极简主义设计哲学，通过优先级分层和渐进披露，确保用户在任何场景下都能快速完成任务。

#### 2.1.1 P0/P1/P2 优先级分层

所有功能和字段按照重要性进行三层分级：

| 优先级 | 定义 | 设计策略 | 示例 |
|--------|------|----------|------|
| **P0** | 必显/核心功能 | 始终显示，不可隐藏 | 需求 ID、标题、状态、操作按钮 |
| **P1** | 重要/常用功能 | 默认显示，可配置隐藏 | 优先级、负责人、分类标签 |
| **P2** | 可选/增强功能 | 默认隐藏，按需展开 | 创建时间、修改时间、备注 |

**实施原则**：
- P0 功能必须在一次点击内可达
- P1 功能必须在三次点击内可达
- P2 功能可以隐藏在二级菜单或配置中

#### 2.1.2 三次点击原则

**定义**：用户从任何页面出发，最多通过 3 次点击即可到达系统中的任何功能模块。

**实施策略**：
- 扁平化导航结构（最多 5 级）
- 常用功能提供快捷入口
- 支持搜索快速定位
- 面包屑导航快速返回

#### 2.1.3 一个屏幕原则

**定义**：核心操作和关键信息必须在一个屏幕内完整展示，无需滚动或跳转。

**实施策略**：
- 详情页采用左右分栏布局
- 列表页支持自定义字段显示
- 关键操作提供悬浮面板
- 减少页面跳转，使用模态框

#### 2.1.4 渐进披露原则

**定义**：默认展示最简洁的界面，高级功能和复杂选项按需展开。

**实施策略**：
- 高级筛选条件可折叠
- 批量操作默认隐藏
- 详情信息分层展示
- 配置项分组管理

### 2.2 已确定方案

#### 2.2.1 登录界面
- **方案**：简化企业登录（用户名/密码）
- **优先级**：P0
- **说明**：移除不必要的注册流程，企业用户通过管理员分配账号

#### 2.2.2 权限模型
- **方案**：混合模式（角色 + 项目授权）
- **优先级**：P0
- **说明**：基于角色定义基础权限，通过项目授权进行细粒度控制

#### 2.2.3 头像上传
- **方案**：用户自助上传（2MB 限制）
- **优先级**：P1
- **说明**：支持 JPG/PNG 格式，自动压缩优化

#### 2.2.4 需求列表字段
- **方案**：P0 必显/P1 重要/P2 可选
- **优先级**：P0
- **字段分级**：
  - P0：复选框、需求 ID、标题 + 描述预览、分类标签、状态、操作
  - P1：优先级、负责人
  - P2：创建时间、修改时间

#### 2.2.5 用例列表字段
- **方案**：P0 必显/P1 重要/P2 可选
- **优先级**：P0
- **字段分级**：
  - P0：复选框、用例 ID、标题 + 描述预览、执行状态、操作
  - P1：用例类型、优先级、负责人
  - P2：最后执行时间

#### 2.2.6 详情页布局
- **方案**：左右分栏
- **优先级**：P0
- **说明**：左侧主内容区（描述/操作步骤），右侧信息区（属性/关系/历史）

#### 2.2.7 导航树
- **方案**：左侧导航，最多 5 级
- **优先级**：P0
- **说明**：支持展开/收起、拖拽排序、自动编号

#### 2.2.8 导出模板
- **方案**：公有/私有两种可见性
- **优先级**：P2
- **说明**：公有模板全员可用，私有模板仅创建者可见

#### 2.2.9 基线视图
- **方案**：修订视图/纯净视图切换
- **优先级**：P1
- **说明**：修订视图显示变更标记，纯净视图显示最终版本

#### 2.2.10 自定义字段
- **方案**：仅需求和用例支持
- **优先级**：P2
- **说明**：其他模块（如项目、基线）不支持自定义字段

```javascript
function regeneratePaths(parentId, startNumber) {
  const children = getChildren(parentId);
  
  children.forEach((child, index) => {
    const parentPath = parentId ? getNodePath(parentId) : '';
    const newPath = parentPath ? `${parentPath}.${index + 1}` : `${index + 1}`;
    
    updateNodePath(child.id, newPath);
    
    if (child.children.length > 0) {
      regeneratePaths(child.id, index + 1);
    }
  });
}
```

### 7.4 License 绑定

```javascript
function getHardwareId() {
  const mac = os.networkInterfaces().eth0.mac;
  const hostname = os.hostname();
  const platform = os.platform();
  
  return crypto
    .createHash('sha256')
    .update(mac + hostname + platform)
    .digest('hex');
}

function verifyLicense(license) {
  const currentHardwareId = getHardwareId();
  
  if (license.hardwareId !== currentHardwareId) {
    throw new Error('License 与硬件不匹配');
  }
}
```
