# 企业级 SaaS 登录界面设计方案

> 文档版本：v1.0  
> 创建日期：2026-03-16  
> 适用场景：License 授权模式的企业级 SaaS 系统

---

## 一、调研对象分析

### 1.1 Jira (Atlassian)

**登录方式**：
- 统一 Atlassian 账号登录 (id.atlassian.com)
- 支持 Google 等第三方登录
- 企业版支持 SSO 单点登录 (SAML 2.0, OpenID Connect)
- 双因素认证 (2FA) 可选

**License 模式**：
- Cloud 版：按用户数订阅，免费 tier (≤10 用户)
- Data Center：企业私有部署，按用户数授权
- Server：已停止新销售 (2024 年 2 月终止支持)

**试用策略**：
- Cloud 版：免费 tier 永久可用
- 付费版：7 天免费试用
- 试用到期后降级为免费版

---

### 1.2 Confluence (Atlassian)

**登录方式**：与 Jira 共享 Atlassian 账号体系

**License 模式**：
- Cloud 版：按用户数订阅
- 免费 tier：≤10 用户，2GB 存储
- 试用：7 天 Standard/Premium 功能试用

---

### 1.3 ONES.cn (国产研发管理平台)

**版本体系**：
| 版本 | 规模 | 价格 | 部署方式 |
|------|------|------|----------|
| 团队版 | ≤50 人 | 免费 | 公有云 |
| 企业版 | ≥10 人 | 按需订阅 | 公有云 |
| 企业版私有部署 | ≥50 人 | 定制报价 | 私有化 |

**登录特性**：
- 支持企业账号目录同步 (LDAP/AD)
- 私有部署支持本地认证
- 多租户架构 (按组织隔离)

**试用策略**：
- 公有云版：免费试用
- 私有部署：POC 测试 + 正式授权

---

### 1.4 禅道 (ZenTao)

**版本体系**：
| 版本 | 授权模式 | 核心功能 |
|------|----------|----------|
| 开源版 | 永久免费 | 需求/项目/测试/效能管理 |
| 企业版 | 商业授权 | + 工时/甘特图/LDAP/移动端 |
| 旗舰版 | 商业授权 | + 矩阵组织/基线/CMMI |
| IPD 版 | 商业授权 | + IPD 流程/TR 评审/DCP 评审 |

**部署方式**：
- 私有部署 (主流)
- 云禅道 (SaaS)

**License 激活**：
- 商业版需购买授权码
- 授权码绑定域名/服务器
- 支持临时试用授权

### 3.3 License 状态管理

| 状态 | 说明 | 系统行为 |
|------|------|----------|
| 🟢 正式授权 | License 有效 | 全功能可用 |
| 🟡 试用期 | 试用授权中 | 全功能可用，显示倒计时 |
| 🟠 即将过期 | 剩余≤7 天 | 显示续费提醒，可正常使用 |
| 🔴 已过期 | License 失效 | 只读模式，禁止新建/编辑 |
| ⚪ 未激活 | 首次安装 | 引导激活或进入试用 |

## 五、安全加固建议

### 5.1 认证安全

```
┌─────────────────────────────────────────────────────────────────┐
│                      认证安全加固清单                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ✅ 密码策略                                                    │
│     • 最小长度：8 位                                             │
│     • 复杂度要求：大小写 + 数字 + 特殊字符                       │
│     • 密码历史：禁止复用最近 5 次密码                           │
│     • 有效期：90 天强制更换 (可配置)                            │
│                                                                 │
│  ✅ 登录保护                                                    │
│     • 失败锁定：5 次失败后锁定 15 分钟                          │
│     • 验证码：连续失败 3 次触发图形验证码                       │
│     • IP 限制：可配置允许登录的 IP 范围                         │
│     • 设备指纹：记录常用设备，异常设备告警                      │
│                                                                 │
│  ✅ 会话管理                                                    │
│     • 会话超时：30 分钟无操作自动登出                           │
│     • 单点登录：支持 SSO，可配置是否允许多地登录                │
│     • 会话令牌：使用 JWT，支持刷新机制                          │
│                                                                 │
│  ✅ 双因素认证 (2FA)                                            │
│     • 支持 TOTP (Google Authenticator)                         │
│     • 支持短信验证码                                            │
│     • 支持邮件验证码                                            │
│     • 支持硬件密钥 (FIDO U2F)                                  │
│     • 管理员强制开启，普通用户可选                             │
│                                                                 │
│  ✅ 传输安全                                                    │
│     • 强制 HTTPS (HSTS)                                        │
│     • TLS 1.3 优先                                              │
│     • 敏感信息加密传输                                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.3 企业级安全特性

| 特性 | 说明 | 适用版本 |
|------|------|----------|
| LDAP/AD 集成 | 对接企业目录服务 | 企业版+ |

## 七、总结与建议

### 7.1 核心设计要点

1. **统一认证入口**：支持多种登录方式，但保持界面简洁
2. **多租户支持**：企业用户可能属于多个组织，需清晰选择
3. **License 灵活激活**：在线/离线/试用三种模式
4. **试用转正式**：平滑过渡，数据保留，提醒及时
5. **安全优先**：从界面到后端全链路安全加固

### 7.2 差异化建议

针对 License 授权模式的系统，建议：

- **首次登录即引导激活**：避免用户困惑
- **License 状态可视化**：在界面显著位置展示
- **到期预警机制**：多级提醒，避免突然中断
- **离线激活简化**：提供详细指引和工具支持
- **试用数据保留**：转正后数据无缝延续
