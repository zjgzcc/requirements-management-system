## 1. 列表设计

### 1.1 Jira - 问题列表

**字段布局特点：**
- 采用表格视图，核心字段：问题类型、摘要、状态、经办人、优先级
- 支持自定义列显示/隐藏
- 每行左侧提供复选框用于批量操作
- 问题键号（如 ENG-2085）作为唯一标识前置显示

**筛选方式：**
- 顶部固定筛选栏：项目、状态、经办人、标签
- 支持保存自定义筛选器（JQL 查询）
- 快速筛选：我的问题、最近查看、打开的问题
- 高级筛选支持布尔逻辑组合

**操作收敛：**
- 行内操作：悬停显示「更多」按钮（...）
- 批量操作：选中后顶部浮现操作栏
- 右键菜单支持快捷状态流转
- 拖拽支持：状态列可直接拖拽变更

### 1.2 Linear - 问题列表

**字段布局特点：**
- 极简设计，仅显示：标题、状态、标签
- 状态采用彩色圆点标识，减少视觉噪音
- 支持分组视图（按状态/经办人/项目）
- 每行右侧显示更新时间（相对时间）

**筛选方式：**
- 左侧预设视图：Backlog、Todo、In Progress、Done
- 支持自定义筛选条件保存
- 键盘快捷键快速筛选（Cmd/Ctrl + K）
- 智能筛选：自动识别关键词

**操作收敛：**
- 单击行进入详情，避免过多行内按钮
- 悬停显示快速操作：分配、状态变更
- 批量编辑支持
- 拖拽排序和状态变更

### 1.3 Notion - 数据库视图

**字段布局特点：**
- 多视图支持：表格、看板、列表、日历、画廊
- 列类型丰富：文本、数字、单选、多选、人员、日期、公式
- 支持列宽拖拽调整
- 属性可折叠/展开

**筛选方式：**
- 顶部筛选栏支持多条件组合
- 视图级别筛选（每个视图独立配置）
- 支持筛选条件保存为独立视图
- 搜索支持全文检索

**操作收敛：**
- 行内「+」号快速添加
- 右键菜单提供完整操作集
- 拖拽支持：行排序、跨视图移动
- 批量操作：多选后顶部浮现工具栏

## 2. 详情页设计

### 2.1 Jira - 问题详情页

**布局结构：**
```
┌─────────────────────────────────────────────────────┐
│  [返回]  问题摘要标题                    [编辑] [...] │
├──────────────────┬──────────────────────────────────┤
│                  │  描述                            │
│  左侧边栏        │  ─────────────────────────────   │
│  ─────────────   │  [富文本编辑器]                  │
│  状态            │                                  │
│  经办人          │  活动记录                        │
│  优先级          │  ─────────────────────────────   │
│  标签            │  [评论输入框]                    │
│  关联问题        │  评论列表（按时间排序）          │
│  附件            │                                  │
└──────────────────┴──────────────────────────────────┘
```

**信息层级：**
- 顶部：问题标识 + 摘要（最醒目）
- 左侧：元数据（状态、人员、优先级）
- 右侧主区域：描述 + 活动流
- 底部：关联信息（子任务、链接问题）

**渐进披露：**
- 描述区域支持折叠/展开
- 活动记录默认显示最近 10 条，「查看更多」加载历史
- 高级字段默认隐藏，点击「显示更多字段」展开
- 附件预览悬停展开

### 2.2 Linear - 问题详情页

**布局结构：**
```
┌─────────────────────────────────────────────────────┐
│  ← 返回列表                                          │
│                                                      │
│  ENG-2085  Reduce UI flicker during autonomy...     │
│                                                      │
│  ┌────────────────┐  ┌──────────────────────────┐   │
│  │                │  │  Description             │   │
│  │  Properties    │  │  ─────────────────────   │   │
│  │  ───────────   │  │  [Markdown 编辑器]        │   │
│  │  Status        │  │                          │   │
│  │  Assignee      │  │  Activity                │   │
│  │  Labels        │  │  ─────────────────────   │   │
│  │  Parent        │  │  [评论...]               │   │
│  │  ...           │  │                          │   │
│  └────────────────┘  └──────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

**设计特点：**
- 左右分栏，左侧属性固定，右侧内容可滚动
- 属性区域支持快速编辑（点击即编辑）
- 描述支持 Markdown 实时预览
- 活动流时间线设计，清晰展示变更历史

### 2.3 Notion - 页面详情

**布局结构：**
- 全宽内容区，最大化阅读体验
- 顶部封面图 + 图标（可选）
- 标题下方支持属性元数据展示
- 块级编辑器，自由拖拽重组

**信息层级：**
- 标题 H1 最醒目
- 正文内容层次分明（H2/H3/段落）
- 侧边栏显示页面树和大纲
- 底部显示元信息（创建时间、最后编辑）

## 3. 导航设计

### 3.1 左侧导航树

**Jira 导航：**
```
┌─────────────────────┐
│  [搜索框 Cmd+K]     │
├─────────────────────┤
│  🏠 首页            │
│  📋 项目            │
│    ▶ 项目 A         │
│    ▶ 项目 B         │
│  🔍 筛选器          │
│  📊 仪表板          │
│  👥 人员            │
│  ⚙️ 设置            │
└─────────────────────┘
```

**设计特点：**
- 固定左侧，宽度可调节（240px-320px）
- 支持折叠/展开，节省空间
- 当前选中项高亮显示
- 支持拖拽重排序
- 键盘导航支持（上下键 + Enter）

**Notion 导航：**
- 树形结构，支持无限层级
- 页面可拖拽重组层级
- 收藏页面置顶显示
- 最近访问自动记录
- 快速跳转：Cmd/Ctrl + P 唤起页面搜索

### 3.2 面包屑导航

**使用场景：**
- 深层级页面定位
- 快速返回上级
- 展示当前位置上下文

**设计模式：**
```
首页 / 项目 / 项目 A / 问题 / ENG-2085
```

**交互细节：**
- 每级可点击跳转
- 悬停显示完整标题（截断时）
- 最后一级不可点击（当前位置）
- 移动端自动折叠中间层级

### 3.3 快速跳转

**Linear 命令面板（Cmd+K）：**
- 全局搜索：问题、项目、人员
- 快速操作：创建问题、切换视图
- 导航跳转：直接跳转到任意页面
- 支持模糊匹配

**Notion 快速跳转（Cmd/Ctrl + P）：**
- 页面搜索
- 最近访问列表
- 收藏页面快捷入口
- 支持创建新页面

## 4. 表单设计

### 4.1 必填/选填标识

**设计模式：**
- 必填字段：标签后加红色星号 `*` 或文字「必填」
- 选填字段：标签后加灰色「可选」或无标识
- 分组标题：清晰区分不同信息区块

**Jira 示例：**
```
摘要 *
[_________________________]

描述
[_________________________]
[_________________________]

经办人 (可选)
[选择人员 ▼]

优先级 *
[高 ▼]
```

### 4.2 实时验证

**验证时机：**
- 失焦验证（onBlur）：用户离开输入框时
- 输入验证（onChange）：实时反馈（适用于格式校验）
- 提交验证：最终完整性检查

**反馈形式：**
- 成功：绿色对勾图标
- 错误：红色文字说明 + 边框高亮
- 警告：黄色提示（非阻断性）

**Linear 示例：**
- 标题长度限制：实时显示字符计数（如 0/256）
- 重复检测：输入时检测相似问题
- 格式校验：URL、邮箱格式实时提示

### 4.3 错误提示

**设计原则：**
- 就近原则：错误信息显示在对应字段下方
- 明确描述：说明具体错误原因和修复方法
- 非阻断性：允许用户继续填写其他字段

**错误提示示例：**
```
❌ 该字段不能为空
❌ 请输入有效的邮箱地址
❌ 标题长度不能超过 256 个字符
⚠️ 已存在相似问题，请确认是否重复
```

## 5. 交互细节

### 5.1 悬停提示（Tooltip）

**使用场景：**
- 图标按钮功能说明
- 截断文本完整显示
- 复杂操作引导

**设计要点：**
- 延迟显示（300-500ms），避免闪烁
- 跟随鼠标位置，避免遮挡
- 暗色背景 + 白色文字，高对比度
- 支持富文本（图标 + 文字 + 快捷键提示）

**示例：**
```
[📎] ← 悬停显示「添加附件 (A)」
[👤] ← 悬停显示「分配经办人」
```

### 5.2 加载动画

**类型选择：**
- 骨架屏（Skeleton）：内容加载，保持布局稳定
- 旋转加载器（Spinner）：操作等待
- 进度条（Progress Bar）：可量化进度
- 脉冲动画（Pulse）：内容更新提示

**Linear 加载策略：**
- 首屏骨架屏，快速呈现页面结构
- 列表项懒加载，滚动时逐步加载
- 乐观更新：操作立即反馈，后台同步

### 5.3 过渡效果

**页面切换：**
- 淡入淡出（Fade）：通用场景
- 滑动（Slide）：层级导航
- 缩放（Scale）：模态框弹出

**列表动画：**
- 新增项：从上滑入 + 淡入
- 删除项：向右滑出 + 淡出
- 排序变更：平滑位置过渡

**微交互：**
- 按钮悬停：轻微放大（1.02-1.05 倍）
- 点击反馈：轻微缩小（0.98 倍）
- 状态变更：颜色渐变过渡

## 6. 风险防控

### 6.1 二次确认

**需要确认的场景：**
- 删除操作（不可恢复）
- 批量操作（影响多条数据）
- 状态回退（可能影响流程）
- 离开未保存页面

**确认对话框设计：**
```
┌─────────────────────────────────────┐
│  ⚠️  确认删除？                     │
│                                     │
│  此操作不可恢复，确定要删除这个问题 │
│  吗？                               │
│                                     │
│            [取消]  [确认删除]       │
└─────────────────────────────────────┘
```

**设计要点：**
- 危险操作使用红色按钮
- 明确说明后果
- 默认焦点在「取消」按钮（防止误触）
- 支持快捷键（Enter 确认，Esc 取消）

### 6.2 自动保存

**保存策略：**
- 定时保存：每 30 秒自动保存
- 失焦保存：离开输入框时保存
- 操作保存：特定操作触发保存

**状态提示：**
```
✓ 已保存          (绿色对勾)
⏳ 正在保存...    (旋转图标)
❌ 保存失败       (红色警告 + 重试按钮)
```

**Notion 实现：**
- 实时保存，每次按键后 debounce 500ms
- 顶部显示保存状态
- 网络断开时本地缓存，恢复后同步

### 6.3 版本历史

**功能设计：**
- 自动记录每次保存的版本
- 支持版本对比（差异高亮）
- 支持恢复到任意历史版本
- 显示版本元信息（时间、操作人）

**版本历史面板：**
```
┌─────────────────────────────┐
│  版本历史                   │
├─────────────────────────────┤
│  ● 当前版本                 │
│  ○ 2 小时前 - 陈先生        │
│  ○ 昨天 15:30 - 李女士      │
│  ○ 昨天 10:00 - 陈先生      │
│  ○ 3 天前 - 系统自动保存    │
└─────────────────────────────┘
```

## 8. 侵权风险评估

### 8.1 风险等级划分

| 风险等级 | 定义 | 处理方式 |
|----------|------|----------|
| 🔴 高风险 | 直接使用受版权保护的素材（图标、图片、代码） | 严格禁止 |
| 🟡 中风险 | 高度相似的视觉设计（配色、布局、交互） | 需差异化改造 |
| 🟢 低风险 | 通用设计模式（行业惯例） | 可安全借鉴 |

### 8.2 具体风险评估

#### 列表设计
- **风险等级：🟢 低风险**
- **说明：** 表格列表是通用 UI 模式，字段布局、筛选方式属于功能设计，不受版权保护
- **建议：** 避免直接复制 Jira/Linear 的图标和配色方案

#### 详情页布局
- **风险等级：🟢 低风险**
- **说明：** 左右分栏是常见布局模式，信息层级组织方式不受保护
- **建议：** 可借鉴结构，但视觉风格需自主设计

#### 导航设计
- **风险等级：🟢 低风险**
- **说明：** 左侧导航树、面包屑是行业标准模式
- **建议：** 命令面板的交互逻辑可借鉴，但视觉实现需独立设计

#### 表单设计
- **风险等级：🟢 低风险**
- **说明：** 必填标识、实时验证是通用交互模式
- **建议：** 验证提示文案需自主撰写

#### 交互细节
- **风险等级：🟡 中风险**
- **说明：** 特定动画曲线、过渡效果可能体现独特设计风格
- **建议：** 参考动画类型，但调整时长、曲线参数

#### 品牌元素
- **风险等级：🔴 高风险**
- **说明：** Logo、专属图标、品牌配色受商标和版权保护
- **建议：** 严格禁止使用，必须自主设计

### 8.3 合规建议

1. **只借鉴模式，不复制实现**
   - 学习设计思路和交互逻辑
   - 视觉风格自主设计

2. **使用开源设计系统**
   - Material Design（Google）
   - Carbon Design System（IBM）
   - Primer（GitHub）
   - 这些系统明确允许商业使用

3. **图标使用规范**
   - 使用开源图标库（Feather Icons、Heroicons、Tabler Icons）
   - 避免直接使用 Jira/Linear 的专属图标

4. **文案自主撰写**
   - 错误提示、引导文案需原创
   - 避免直接复制产品文案

5. **差异化设计**
   - 在借鉴基础上加入创新元素
   - 形成独特的产品风格

## 9. 我们的改进建议

### 9.1 针对医疗器械行业的特殊优化

### B. 设计原则总结

1. **一致性**：同一产品内保持设计语言统一
2. **效率优先**：减少操作步骤，支持快捷键
3. **渐进披露**：复杂功能按需展示
4. **即时反馈**：所有操作都有明确反馈
5. **容错设计**：支持撤销、提供二次确认
6. **无障碍**：照顾所有用户的使用需求
