### 1.2 用户画像

#### 用户画像 1：医疗器械研发工程师
```
姓名：张工
年龄：35 岁
职业：医疗器械研发工程师
公司：某医疗器械上市公司

使用场景：
- 将扫描版技术标准（GB/T、YY/T）转换为可搜索 PDF
- 查阅历史检测报告
- 整理研发文档

核心需求：
- 数据不能外泄（公司机密）
- 中文识别准确率高
- 支持批量处理（一次处理 10+ 个文件）
- 符合行业合规要求（可追溯、可审计）

痛点：
- 现有在线 OCR 工具不敢用（怕数据泄露）
- Adobe Acrobat 太贵（公司不愿付费）
- OCRmyPDF 没有图形界面（同事不会用）
```

#### 用户画像 2：企业文档管理员
```
姓名：李姐
年龄：42 岁
职业：文档管理员
公司：某制造企业

使用场景：
- 归档历史纸质文档
- 批量扫描合同、报告
- 建立可搜索的文档库

核心需求：
- 批量上传和处理（一次 50+ 文件）
- 处理速度快（不影响工作进度）
- 简单易用（不需要培训）
- 价格能接受（预算有限）

痛点：
- 扫描文件太多，一个一个处理太慢
- 有些 OCR 工具识别后格式错乱
- 找不到免费又好用的工具
```

```
核心需求：
```

### 3.2 痛点场景还原

#### 场景 1：数据泄露担忧
```
张工（医疗器械研发）：
"我们有很多技术标准文档和检测报告，都是扫描版的 PDF。
想要转换成可搜索的方便查阅，但是不敢用网上的 OCR 工具，
怕数据泄露。Adobe Acrobat 倒是可以，但是公司不愿意花那个钱。"

→ 需求：本地部署 + 数据安全 + 价格合理
```

#### 场景 2：批量处理效率低
```
李姐（文档管理员）：
"我每天要处理几十上百个扫描文件，一个一个上传太慢了。
有些工具说支持批量，但是上传 10 个文件就卡住了。
要是能一次上传几十个，然后自动处理就好了。"

→ 需求：批量上传 + 队列处理 + 稳定性
```

#### 场景 3：格式错乱问题
```
陈教授（学术研究）：
"我有很多历史文献，里面有表格和公式。
用了一些 OCR 工具后，文字是识别出来了，但是表格结构全乱了，
公式也变成乱码，根本没法用。"

→ 需求：表格识别 + 公式识别 + 格式保持
```

## 6. 用户使用场景

### 6.1 典型使用流程

#### 场景 A：单文件处理
```
1. 打开网站
2. 拖拽 PDF 文件到上传区域
3. 点击"开始转换"
4. 等待处理（查看进度条）
5. 下载转换后的 PDF
6. 完成

```

#### 场景 B：批量处理
```
1. 打开网站
2. 选择多个 PDF 文件（10-50 个）
3. 点击"批量上传"
4. 系统自动队列处理
5. 可以离线等待
6. 完成后邮件通知
7. 批量下载

期望时间：< 30 分钟（50 个文件）
```

```
1. 注册账号
```
