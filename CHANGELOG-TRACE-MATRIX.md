### 1. ✅ 矩阵展示性能优化（大数据量分页）

#### 后端 API 变更
- **接口**: `GET /api/trace-matrix`
- **新增参数**:

  ```json
      "page": 1,
      "pageSize": 20,
      "totalItems": 100,
      "totalPages": 5,
      "hasNextPage": true,
      "hasPrevPage": false
    }
  }
  ```

#### 前端功能
- ✅ 分页控件（上一页/下一页/页码跳转）
- ✅ 页码显示（当前页/总页数）
- ✅ 快速跳转到指定页
- ✅ 每页固定显示 20 条需求（可配置）

### 2. ✅ 批量关联功能

#### 后端 API 新增
- **接口**: `POST /api/traces/batch`
- **请求体**:
  ```json
  {
    "traces": [
      { "requirementId": "req1", "testcaseId": "tc1" },
      { "requirementId": "req1", "testcaseId": "tc2" }
    ]
  }
  ```
- **响应**:
  ```json
  {
    "success": true,
    "message": "批量创建完成：成功 10 条，重复 2 条，失败 0 条",
    "data": {
      "success": [...],
      "failed": [...],
      "duplicates": [...]
    }
  }
  ```

#### 前端功能
- ✅ 批量选择需求（全选/单选）
- ✅ 批量关联模态框
- ✅ 多选用例（支持 Ctrl/Cmd 多选）
- ✅ 批量操作工具栏
- ✅ 实时显示已选择数量

#### 使用流程
1. 切换到「追踪矩阵」标签页
2. 点击「批量关联」按钮
3. 勾选需要关联的需求（或使用全选）
4. 在模态框中选择一个或多个用例
5. 点击「确认关联」

### 3. ✅ 智能推荐关联

#### 后端 API 新增
- **接口**: `GET /api/traces/recommend`
- **参数**:
  - `projectId`: 项目 ID（必填）
  - `requirementId`: 指定需求 ID（可选，不传则推荐所有）
- **响应**:
  ```json
  {
    "success": true,
    "data": [
      {
        "requirementId": "req1",
        "requirementDesc": "需求描述",
        "testcaseId": "tc1",
        "testcaseSteps": "用例步骤",
        "matchScore": 5,
        "matchedKeywords": ["登录", "验证", "用户"]
      }
    ]
  }
  ```

#### 算法说明
- **关键词提取**: 从需求和用例的文本中提取关键词（中文分词简化版）
- **停用词过滤**: 自动过滤常见停用词（的、了、是、在等）
- **匹配度计算**: 基于共同关键词数量
- **排序**: 按匹配度降序排列
- **限制**: 最多返回 50 条推荐

#### 前端功能
- ✅ 智能推荐模态框
- ✅ 推荐列表展示（需求、用例、匹配度、关键词）
- ✅ 一键接受推荐（建立追踪关系）
- ✅ 实时刷新推荐
